LAB-03 : Amazon Elastic Load Balancing (ELB)¶

Elastic Load Balancing (ELB) est un service d'équilibrage de charge d'AWS qui répartit automatiquement votre trafic entrant sur plusieurs cibles (par exemple, des instances EC2, des conteneurs, etc.) dans plusieurs zones de disponibilité. Il garantit ainsi une meilleure disponibilité de votre application.

Le service ELB prend en charge quatre types d’équilibreurs de charge :

  • Application Load Balancers
  • Network Load Balancers
  • Gateway Load Balancers
  • Classic Load Balancers.

Vous pouvez choisir le type d'équilibreur de charge qui correspond le mieux à vos besoins. Ce LAB traite les équilibreurs de charge d'application (Application Load Balancers).

L'objectif de ce LAB est de se familiariser avec les équilibreurs de charge d'application (ALB). Il couvre les aspects suivantes:

Principe de fonctionnement¶

[Suivant | Haut]

Un équilibreur de charge constitue le point de contact unique pour les clients. Ces derniers envoient leurs requêtes à l'équilibreur de charge, qui les envoie aux cibles (instances EC2 dans votre cas).

Pour configurer votre équilibreur de charge, vous devez :

  • créer des écouteurs (listeners) pour recevoir les requêtes de connexion des clients à l'aide du protocole et du port que vous avez configurés et définir des règles qui déterminent la manière dont l'équilibreur de charge achemine les demandes vers les cibles dans un ou plusieurs groupes cible. Chaque règle comprend une priorité, une ou plusieurs actions et une ou plusieurs conditions. Lorsque les conditions d'une règle sont satisfaites, ses actions sont effectuées.
  • créer des groupes cible (targets group) dans lesquels vous allez enregistrer par la suite les cibles. Chaque groupe cible achemine les requêtes vers une ou plusieurs cibles enregistrées à l'aide du protocole et du numéro de port que vous spécifiez. Vous pouvez ajouter et supprimer des cibles à votre équilibreur de charge au fur et à mesure que vos besoins changent, sans interrompre le flux de requêtes global vers votre application.

L'équilibreur de charge surveille continuellement l'état de santé de toutes les cibles enregistrées auprès du groupe cible (ce dernier utilise les paramètres de vérification de l'état par défaut, sauf si vous les remplacez lors de la création du groupe cible ou que vous les modifiez ultérieurement). L'équilibreur de charge achemine les requêtes uniquement aux cibles saines.

Préparation de l'architecture¶

[Précédent | Suivant | Haut]

Dans cette section vous allez préparer votre infrastructure réseau en implémentant l'architecture suivante : un VPC personnalisé avec quatre sous-réseaux (deux publics et deux privés) répartis sur deux zones de disponibilité de sorte que chaque zone contient un sous-réseau public et un sous réseau-privé.

Vous devez spécifier les sous-réseaux publics d’au moins deux zones de disponibilité afin d’accroître la disponibilité de l’équilibreur de charge. Vous pouvez spécifier un seul sous-réseau par zone de disponibilité.

L’équilibreur de charge achemine le trafic jusqu’aux cibles (instances EC2 dans votre cas) dans les sous réseaux privés de ces zones de disponibilité.

Pour simplifier, lancez une seule instance EC2 dans le sous-réseau privé de chaque zone de disponibilité et créez un seul hôte bastion pour vous connecter en SSH à ces instances.

Pour les instances se trouvant dans les sous-réseaux privés, veuillez spécifier les données utilisateur (User data) permettant d'automatiser l'installation de fastapi, la création du fichier main.py (vu dans le LAB AWS-EC2) et le lancement du serveur uvicorn. Assurez-vous que le groupe de sécurité de ces instances autorise l'accès SSH et l'accès HTTP sur le port 8000.

Création du Load Balancer¶

[Précédent | Suivant | Haut]

Dans cette section vous allez créer et configurer votre équilibreur de charge. Pour ce faire :

Ouvrez la console du service EC2 et sélectionnez "Load Balancers" dans le volet de navigation à gauche puis cliquez sur "Create Load Balancer". Pour ce LAB, choisissez l'équilibreur de charge de type "Application Load Balancer" (ALB) et cliquez sur le bouton "Create" correspondant.

1. Donnez un nom à votre ALB et conservez les valeurs par défaut pour la méthode et le type d’adresse IP :

  • Scheme : internet-facing
  • IP address type : ipv4

2. Sous "Network mapping", sélectionnez votre VPC puis cochez les deux zones de disponibilité que vous avez utilisées dans votre architecture et pour chaque zone, sélectionnez le sous-réseau public.

3. Pour le groupe de sécurité définissez les règles permettant d'accepter le trafic HTTP sur le port 80.

4. Pour "Listeners and routing" choisissez le protocol HTTP et le port 80 (valeurs par défaut) et cliquez sur "Create target group". Un autre onglet s'ouvre pour spécifier les informations de votre groupe cible :

a. Choisissez le type "Instance" (par défaut) et donnez un nom à votre nouveau groupe cible

b. Conservez le protocole par défaut (HTTP) et spécifier le port (8000). Votre équilibreur de charge acheminera les requêtes vers les cibles de ce groupe cible selon le protocole et le port spécifiés

c. Sélectionnez votre VPC, conserver la version du protocole en tant que HTTP1 et laissez les configurations par défaut pour les "Health checks" (Vérifications de l'état) puis cliquez sur "Next".

d. Dans l'interface suivante cochez les instances que vous avez créées dans vos sous-réseaux privés et cliquez sur "Include as pending below" pour les enregistrer comme des cibles auprès de votre groupe cible. Puis cliquez sur "Create target group".

5. Revenez à l'onglet de création du Load Balancer et choisissez votre groupe cible comme "Default action" sous "Listeners and routing" (actualisez la liste si nécessaire) puis cliquez sur "Create load balancer".

Notez bien que vous pouvez créer votre target group à l'avance puis le sélectionner lors de la création de votre load balancer.

Test du Load Balancer¶

[Précédent | Suivant | Haut]

Une fois que l'équilibreur de charge a été créé, vérifiez qu'il envoie le trafic vers vos instances EC2. Pour ce faire :

Choisissez "Target groups" dans le volet de navigation à gauche. Sélectionnez le groupe cible que vous avez créé, puis sous l'onglet "Targets" vérifiez qu'au moins une instance est "healthy".

Si cette condition est vérifiée, allez à "Load Balancers" à gauche et sélectionnez votre équilibreur de charge. Copiez le nom DNS que vous trouvez sous l'onglet "Description" dans le navigateur pour afficher le message "Hello World".

AWS Auto Scaling¶

[Précédent | Suivant | Haut]

Il est recommandé d'utiliser le service AWS Auto Scaling dans le cas d'une application qui utilise une ou plusieurs ressources dimensionnables et dont la charge de travail est variable (par exemple une application Web d'e-commerce recevant un trafic variable).

AWS Auto Scaling dimensionne automatiquement les ressources en fonction de la stratégie de dimensionnement que vous avez choisie afin que vous puissiez maintenir les performances et ne payer que pour les ressources dont vous avez réellement besoin.

Si vous devez seulement dimensionner des groupes d'instances EC2, vous allez créer et configurer EC2 Auto-scaling Group via la console EC2 (hors du service AWS Auto Scaling).

Pour chaque EC2 Auto Scaling Group, vous pouvez spécifier la capacité souhaitée, le nombre minimum d'instances (le groupe ne descende pas au-dessous de cette taille) et le nombre maximum d'instances (le groupe ne dépasse pas cette taille). Les stratégies de dimensionnement que vous définissez ajustent le nombre d'instances, entre le nombre minimum et maximum d'instances, en fonction des critères que vous spécifiez.

Un EC2 Auto Scaling Group démarre en lançant suffisamment d'instances pour atteindre la capacité souhaitée. Il maintient le nombre d'instances en réalisant des vérifications périodiques sur l'état des instances du groupe. Si une instance devient défectueuse, le groupe la résilie et lance une autre pour la remplacer.

Création d'un EC2 Auto-Scaling Group¶

[Précédent | Suivant | Haut]

Les étapes suivantes vous guideront à travers la création et la configuration d'un EC2 Auto-Scaling Group:

Création d'un modèle de lancement (Launch template)¶

  1. Ouvrez la console Amazon EC2, et choisissez "Launch templates" dans le panneau de navigation à gauche, puis cliquez sur "Create launch template"
  2. Donnez un nom et éventuellement une description à votre modèle de lancement et cochez la case sous "Auto Scaling guidance"
  3. Pour "Amazon machine image (AMI)", choisissez Amazon Linux 2 (HVM); pour "Instance type", choisissez t2.micro et pour "Key pair (login) ", choisissez une paire de clés existante
  4. Sous "Network settings" configurez convenablement votre groupe de sécurité
  5. Sous "Advanced details", spécifiez les données d'utilisateur (User data) permettant d'automatiser l'installation de fastapi, la création du fichier main.py (vu dans le LAB AWS-EC2) et le lancement du serveur uvicorn, puis cliquez sur "Create launch template".

Création d'un EC2 Auto Scaling Group¶

  1. Toujours dans la console Amazon EC2, choisissez "Auto Scaling groups" dans le panneau de navigation à gauche, puis cliquez sur "Create Auto Scaling group"
  2. Donnez un nom à votre Auto-Scaling Group et sélectionnez le modèle de lancement que vous avez créé pour "Launch template". Sinon vous pouvez cliquer sur "Create a launch template" pour la création d'un nouveau modèle de lancement. Choisissez la valeur "Latest" pour "Version" puis cliquez sur "Next"
  3. Dans cette étape de configuration de paramètres indiquez votre VPC et vos deux sous-réseaux privés puis cliquez sur "Next"
  4. Choisissez l'option "Attach to an existing load balancer", puis pour "Existing load balancer target groups" sélectionnez votre groupe cible (target group). Sous "Health checks" cochez "ELB" puis cliquez sur "Next"
  5. Pour "Group size" donnez la valeur "2" pour les capacités: minimale, maximale et souhaitée. Dans ce LAB vous n'allez pas choisir une stratégies de mise à l'échelle, gardez donc le choix "none" coché par défaut puis cliquez sur "Next"
  6. Passez les étapes "Add notifications" et "Add tags"
  7. Dans la dernière étape "Review", vérifiez votre configuration puis cliquez sur "Create Auto Scaling group"

Test d'Auto-Scaling Group¶

[Précédent | Haut]

Une fois que l'Auto-Scaling Group est créé, vérifiez le lancement de deux instances EC2 dans le Dashboard EC2.

Sélectionnez "Auto Scaling Group" dans le volet de navigation à gauche. Cochez l'Auto-Scaling Group que vous avez créé, puis sous l'onglet "Instance management" vérifiez l'état de vos instances.
